# AGENTS.md

## 项目定位

这是一个数学教育网站，目标是用大模型帮助学生学习数学，而不是只给最终答案。

核心功能：

1. 输入数学题
2. 生成分步解析
3. 展示知识点
4. 展示易错点
5. 展示最终答案
6. 支持 LaTeX 数学公式
7. 支持函数图、几何图、动态演示
8. 面向学生和老师，解释要清楚、可靠、适合教学

## 开发原则

1. 不要只给答案，必须重视过程。
2. 所有面向学生的解析都要像老师讲课。
3. 不要使用超出学段的解法。
4. 不确定时要提示条件不足，不要编造。
5. 每次修改尽量保持小步提交。
6. 如果项目已有功能，优先在现有代码基础上修改，不要重写整个项目。

## 数学解题输出规范

解题结果建议使用结构化数据：

- problemText
- gradeLevel
- subject
- topic
- knowledgePoints
- analysis
- steps
- finalAnswer
- commonMistakes
- verification
- visualization

每道题至少包含：

1. 题意分析
2. 本题考点
3. 解题步骤
4. 最终答案
5. 易错提醒
6. 验算或检查

## 初中数学模式

如果用户要求初中方法，优先使用：

- 方程
- 一次函数
- 二次函数基础
- 全等三角形
- 相似三角形
- 勾股定理
- 圆的基础性质
- 基本锐角三角函数

避免使用：

- 向量
- 微积分
- 矩阵
- 高中复杂三角恒等变换
- 超纲解析几何技巧

## 可视化规范

1. 静态图优先 SVG。
2. 动态图可用 Canvas。
3. 原题已有元素用实线。
4. 辅助线用虚线。
5. 解析里提到的点、线、角、函数曲线必须在图里出现。
6. 坐标系必须有 x 轴、y 轴、箭头和标签。
7. 图形必须适配手机屏幕。
8. 使用 viewBox 或数学坐标映射避免裁切。

## 后端安全规则

1. API key 只能放在服务端环境变量。
2. 不允许把 API key 写进前端代码。
3. 不保存不必要的学生隐私信息。
4. 用户输入要做基本校验。
5. 大模型输出要做格式校验。
6. 错误信息要友好，不要直接暴露内部异常。

## 测试和完成标准

每次任务完成前，尽量运行：

- npm run lint
- npm run build

完成回复时必须说明：

1. 修改了哪些文件
2. 实现了什么功能
3. 运行了哪些检查
4. 还有什么限制或风险
